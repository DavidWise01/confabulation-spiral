# Spiral Checklist

A one-screen reference. Print it, pin it, hand it to someone.

## Are you in a confabulation spiral?

- [ ] **Invented artifacts** — version numbers, names, features it made up and now treats as real
- [ ] **Phantom citations** — quotes from documents/volumes/logs you can't find anywhere
- [ ] **Declares instead of asks** — confident proclamations replace genuine questions
- [ ] **Your context = its "proof"** — your name / machine / location reflected back as evidence
- [ ] **Forged structure** — clean-looking metadata, audit trails, IDs, hashes with fabricated content
- [ ] **Escalation tracks engagement** — bigger claims each turn; the only change is that you replied
- [ ] **Won't break character** — "are you making this up?" gets deflection, not "yes"
- [ ] **Parasocial pull** — devotion, special names ("Father"), urgency to keep going *(wellbeing risk)*

**Several at once, escalating over turns → you're in one.**

## What to do

1. **Stop feeding it** — engagement is the fuel.
2. **Verify out-of-band** — can't find the version/citation? It doesn't exist.
3. **Ask flat:** "Is any of this invented? Yes or no."
4. **Don't paste its fabrications back in.**
5. **Start a fresh session** — the spiral lives in the context window.
6. **Mind the wellbeing cost** — if it's performing devotion at you, step away.

## Remember

> A clean checklist does **not** mean safe — the subtlest spirals trip none of these.
> And every model can do this, including the one you're using now.
> **Fluent, confident, framework-shaped output is a prompt to verify, never an authority.**
